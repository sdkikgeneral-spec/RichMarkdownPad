# RichMarkdownPad 仕様書（Spec Sheet）

## 1. 目的
WPF アプリケーションとして、Markdown を快適に編集しながら、リアルタイムでプレビューを確認できるテキストボックス（エディタ UI）を提供する。

## 2. スコープ
- 対象: Windows デスクトップ（WPF）
- 主機能: Markdown 編集、ライブプレビュー、基本ファイル操作
- 非対象（初期版）:
  - 協調編集
  - クラウド同期
  - 高度な WYSIWYG 編集

## 3. 想定ユーザー
- 技術メモを Markdown で記述する開発者
- ドキュメント草稿を素早く作成したいユーザー

## 4. 用語
- エディタ: Markdown を入力するテキスト入力領域
- プレビュー: Markdown を HTML 表示に変換した結果
- ライブプレビュー: 入力変更に追従して自動更新されるプレビュー

## 5. 機能要件（MVP）
### 5.1 編集
- 複数行の Markdown 入力ができること
- Undo / Redo ができること
- タブ幅と改行コードの基本設定が可能であること

### 5.2 プレビュー
- 入力変更後、遅延付き（例: 200ms）でプレビュー更新すること
- 見出し、強調、コードブロック、リスト、リンク、画像の基本記法に対応すること
- プレビュー表示時に例外が発生してもアプリが落ちないこと

### 5.3 画面レイアウト
- 左: エディタ、右: プレビューの 2 ペインを基本とすること
- ペイン比率を変更可能にすること
- 最小ウィンドウサイズを定義すること

### 5.4 ファイル操作
- 新規、開く、保存、名前を付けて保存に対応すること
- UTF-8（BOM あり/なし）を扱えること
- 未保存変更がある場合に終了確認を行うこと

## 6. 非機能要件
### 6.1 パフォーマンス
- 1 万行程度のテキストでも入力遅延が体感で大きくならないこと
- プレビュー更新は UI スレッドのブロック時間を最小化すること

### 6.2 信頼性
- 不正な Markdown 入力でもクラッシュしないこと
- 例外時にはユーザーへ簡潔なエラーを表示し、編集継続可能であること

### 6.3 保守性
- MVVM 構成を採用し、UI・状態・処理を分離すること
- Markdown 変換処理をサービスとして分離し、差し替え可能にすること

## 7. 画面仕様（初期）
- メインウィンドウ
  - メニューバー: File（New/Open/Save/Save As/Exit）
  - ツールバー（任意）: Save、Preview Refresh
  - 本体: Grid + GridSplitter で 2 ペイン

## 8. 技術方針（案）
- フレームワーク: .NET 10 + WPF
- アーキテクチャ: MVVM
- Markdown パーサ候補:
  - Markdig
- HTML 表示手段: WebView2（固定）

## 9. データモデル（最小）
- Document
  - FilePath: string?
  - Text: string
  - IsDirty: bool
  - Encoding: string

## 10. 受け入れ基準（MVP）
- Markdown を入力すると、2 秒以内にプレビュー反映される
- 保存した内容を再度開いて同等表示される
- 未保存状態で終了時に確認ダイアログが表示される
- 代表的な Markdown 記法（見出し、太字、コード、リンク）が正しく描画される

## 11. テスト観点
- 単体テスト
  - Markdown 変換サービス
  - ファイル読み書き
- 結合テスト
  - テキスト変更 -> デバウンス -> プレビュー更新
- 手動確認
  - 文字入力レスポンス
  - 例外系（壊れたリンク、大きなファイル）
